Add Article Help Topics



To create a new KB article, click on the "Create Article" link on the KB home page.

Then complete the following information:

  • Title: Enter a descriptive title of the article.
  • Group: Select the group that owns this article.


  • Viewable By: Select from:
    • Group Members - Only members of the owning group can view this article
    • Public - Any KB user can view the article
    • Editors - Only a KB user with a privilege level of Editor or above can view article
    • Administrator - Only KB Administrators can view article.
  • Type: Select the appropriate type from the list
  • Product: Enter a product name or select from the list
  • Must Read: If set to yes then all users that are a member of this group and have the Must Read flag set for this group are presented with this article on the home page under a special section called Must Read articles. Use this for Articles that you want to enforce all users to read.
  • Custom Field: A custom drop list field may be displayed here if enabled by the Administrator and labeled accordingly
  • Custom Date Field: Another custom field which maybe enabled by the Administrator.
  • Optional Keywords: You can increase the ranking of this article by adding keywords in this box.
  • Related Articles: Add any articles which are related to this article. (You can perform this function only after saving this article).


  • Status: The default status for new articles is Compose. Only Approvers or the Author of the article can view articles that do not have a status of Active.
  • Contacts: Enter Contact name if desired, as appropriate for the content of this KB article. These fields are optional. However persons listed as the contact will see this article listed under "My Articles" on the home page and can also receive notifications of updates.
  • Review By: By default when you save the article, this field will be set for 1 year from the create date or as set by the Administrator.
  • Review Priority: Choose either Low or High. High priority articles needing review will display as a seperate link on the KB home page, this helps bring to attention those articles that should be made available quickly.
  • Expires: Optional date that sets the expiration date of the article. Once expired the article will not be visible from the search page.

Submit For Review button - When an article is in Compose status the submit for Review button is enabled to allow authors to indicate that the article is ready for review. Pressing this button moves the Status to Pending Review which will cause an email notification to Approvers who have this notification type enabled. Once in Technical or Content review status, the article will be displayed on the home page quick links for those that have Approval privilage for the group.

Mark Reviewed button - When an article is in any status other than Compose. Pressing the Mark Reviewed button (Approvers only) will move the article Status to the next value or Active. It also sets the Last Reviewed time stamp and is audited under the Audit tab. Once an Article is Active it is considered "Published" and available to users.

Article Content

The article content can be created using the Article Editor, which is a basic HTML editor or Article content can be imported from a .MHT file or the Article Content can be directed at an uploaded attachment.

Article Editor

  • The Article content editor provides a rich editor window for entering or pasting the content of the article.
  • Hover your mouse over each button icon to see the function of the button.
  • See the topic Modifying Article Content, for more information about using the Content editor.

Import Content

  • The Import Content button allows you to upload a .MHT file which will be converted to HTML and stored as the Article content.
  • MHT files can be produced from Microsoft Word by saving a document in .MHT/MHTML format and can contain images which are embeded in the file. This provides an easier method than taking an existing word document, extracting the images and uploading each image separately.
  • Some formatting problems can occur on complex documents.
  • Note: Do not use this feature for large documents (> 100K) as editing dynamic content for large documents within the browser will cause performance problems with the desktop browser. Instead large documents should be included as an attachment or attachment as content.

Attachment as Content

  • You may upload a Word, MS Excel, PDF file etc as an attachment that will be used as the article content without conversion.
  • In this case the content body of the article is emtpy, and this attachment is viewed automatically embeded in a frame using its native application (Word, Excel, PDF Reader).
  • When you upload an Attachment as Content, the Article Editor will disappear and this attachment will override the content.
  • To update this content, you will need to press the Replace button and upload a newer version of the document. The previous version will be archived.

Article Archives

When ever a change is made to the content or attachments of an article, the complete article and its attachments are archived. Users with Approval or higher privilage can view the archive versions by using the links when viewing the article. When in the article administraion window you can also restore previous archives as the active version (thus undoing any changes made).

The default number of versions to keep is 5, but this can be controlled by an Administrator.
